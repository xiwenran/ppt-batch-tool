# PPT → 笔记制作 一键流水线 Skill

两步合一：PPT 导出图片 → 融景合成，全程自动，输出即可用的笔记素材图。

## CLI 路径

```
python3 ~/ppt-batch-tool/pipeline.py run ...
```

## 输出目录结构

```
输出目录/
  PPT图片/              ← Step1 中间产物（PPT导出的原始PNG）
    课件A/
      1.png  2.png ...
    课件B/
      1.png  2.png ...
  合成图/               ← Step2 最终产物（融景合成完的笔记图）
    课件A/
      1/  1.jpg 2.jpg ...    ← 模板1
      2/  1.jpg 2.jpg ...    ← 模板2
      ...
    课件B/
      ...
```

## 工作流程

### Step 0：确认参数

用户需要提供：
1. **输入**：含 PPT 文件的目录（递归扫描），**或单个 PPT 文件路径**
2. **输出目录**：没有指定时默认用 `输入目录/../笔记制作输出/`

可选参数（不说就用默认值）：
- **模板**：默认全部 22 个；用户说「只用模板1和2」时传 `--templates 1 2`
- **导出页数**：默认 17 页；用户说「只要前10页」时传 `--max-slides 10`

### Step 1：运行流水线

```bash
cd ~/ppt-batch-tool && python3 pipeline.py run \
  --input <PPT文件夹> \
  --output <输出目录> \
  [--templates 1 2 3 ...] \
  [--max-slides 17] \
  [--format JPEG]
```

不指定 `--templates` 时脚本自动获取融景全部可用模板。

### Step 2：报告结果

完成后告诉用户：
- 处理了几个 PPT
- 每个 PPT 导出了几页
- 合成了多少张图（组数 × 模板数 × 页数）
- PPT图片 和 合成图 分别在哪个目录

## 典型用法示例

> 「把这个文件夹的 PPT 一键做成笔记图，存到 Downloads/已制作」

→ `--input <文件夹>`，自动用全部 22 个模板，默认 17 页

> 「把这个 PPT 用模板3制作」（单文件）

→ `--input <具体PPT路径> --templates 3`

> 「只用模板1到5，导出前10页，其余一样」

→ `--templates 1 2 3 4 5 --max-slides 10`

## 注意事项

- macOS 上 PowerPoint 首次运行会弹授权窗口，告知用户点「允许」即可
- **单文件模式不会触发额外授权**：pipeline 不复制文件，直接在原始目录操作（用 `--only-file` 过滤）
- 融景模板需要提前在融景 App 里创建好（标注背景图角点）
- 输出的 PPT图片/ 目录保留，可以单独拿来用（如直接发给融景做其他合成）
